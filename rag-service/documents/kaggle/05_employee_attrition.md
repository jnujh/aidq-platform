## Dataset Overview
- Rows: 1,470 / Columns: 35
- Task: Binary classification (attrition yes/no)
- Domain: Human Resources
- Target variable: Attrition (Yes: 16.1%, No: 83.9%)

## Data Quality Issues Found

### Class Balance
- Attrition rate: 16.1% (237 left, 1,233 stayed)
- Moderate imbalance — better than banking datasets but still skewed
- Small dataset size (1,470) makes minority class even smaller (237)
- Fix: SMOTE or class_weight, be careful with train/test split (stratify)

### Completeness
- Zero missing values — dataset is synthetically generated by IBM
- This is perfect completeness but comes with a caveat: synthetic data may not capture real-world messiness
- Real HR data typically has missing values in: years_at_company (new hires), satisfaction_survey (non-respondents)

### Validity (Constant Features)
- EmployeeCount = 1 for all rows (constant, useless)
- Over18 = "Y" for all rows (constant, useless)
- StandardHours = 80 for all rows (constant, useless)
- These add no information and should be dropped immediately
- Fix: df.drop(columns=['EmployeeCount', 'Over18', 'StandardHours'])

### Consistency (Ordinal Encoding)
- Education: 1=Below College, 2=College, 3=Bachelor, 4=Master, 5=Doctor
- EnvironmentSatisfaction: 1=Low, 2=Medium, 3=High, 4=Very High
- JobInvolvement: 1=Low, 2=Medium, 3=High, 4=Very High
- These are already ordinally encoded — consistent but requires documentation awareness
- Some models treat these as continuous (linear relationship assumed) — may not be appropriate
- Fix: For tree-based models, keep as-is. For linear models, consider one-hot encoding.

### Feature Correlation
- MonthlyIncome and JobLevel: correlation 0.95 (very high)
- TotalWorkingYears and JobLevel: correlation 0.78
- YearsAtCompany and YearsWithCurrManager: correlation 0.77
- Age and TotalWorkingYears: correlation 0.68
- Fix: Keep one of highly correlated pairs, or use tree-based models that handle correlation well

### Outlier Ratio
- MonthlyIncome: right-skewed, range $1,009 to $19,999
- YearsAtCompany: max 40, but median is 5 (right-skewed)
- NumCompaniesWorked: max 9, most values 0-3
- These are not errors — they represent senior employees and long-tenure workers
- Fix: No outlier removal needed, but log-transform for linear models

### Uniqueness
- EmployeeNumber is unique identifier (1,470 unique values = no duplicates)
- No duplicate rows — each employee appears once
- Some employees have very similar profiles but different attrition outcomes — valuable for learning

## Key Insights
- Always check for constant features (zero variance) — easy win for dimensionality reduction
- Synthetic/clean datasets (zero missing) should be noted — production data will be messier
- Ordinal features (1-5 scales) require careful treatment depending on model type
- Small datasets (1,470) with 16% minority = only 237 positive samples — every sample matters
- HR attrition is similar to customer churn but with different key features (satisfaction, overtime, work-life balance)
- MonthlyIncome and JobLevel being 0.95 correlated is expected (salary grades) — domain knowledge helps decide which to keep
